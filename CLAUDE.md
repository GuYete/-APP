# 黑马记账 APP — 产品文档

## 📋 项目概述

- **项目名称**：黑马记账
- **项目目标**：打造一款简洁易用的个人记账应用，帮助用户记录和管理日常花销
- **目标用户**：个人用户，日常消费记录
- **运行平台**：Windows 和 Mac 电脑（跨平台桌面应用）

---

## 🎯 核心功能需求

### 1. 花销记录
- 记录每一笔花销的**金额**（人民币 ¥）
- 记录花销的**日期和时间**
- 选择花销的**分类**（二级分类体系）
- 可添加**备注说明**（可选）
- 支持**删除**已记录的花销

### 2. 花销分类体系（二级分类）

> 用户要求：一级为"大类"，二级为"小类"。以下分类由 Claude 设计，待用户确认。

| 一级分类（大类） | 二级分类（小类） |
|---|---|
| 🍜 **餐饮饮食** | 早餐、午餐、晚餐、零食饮料、外卖、聚餐 |
| 🚗 **交通出行** | 公共交通、打车/网约车、加油/充电、停车费、火车/飞机 |
| 🛒 **购物消费** | 日用品、服饰鞋包、数码产品、家居用品、美妆护肤 |
| 🏠 **住房生活** | 房租/房贷、水电燃气、物业费、维修装修、家居日杂 |
| 🎮 **娱乐休闲** | 电影演出、游戏充值、旅游度假、运动健身、KTV/酒吧 |
| 💊 **医疗健康** | 看病挂号、药品、体检、保健品 |
| 📚 **教育学习** | 书籍、课程培训、文具、考试报名 |
| 🎁 **人情往来** | 礼物、红包、请客吃饭、婚礼份子 |
| 🔧 **其他杂项** | 快递运费、宠物用品、其他 |

### 3. 花销查看
- 按**时间**查看花销列表（按日期倒序，最新的在前面）
- 按**分类**筛选查看
- 按**月份**汇总查看
- 按**金额范围**筛选

### 4. 数据统计
- **月度总支出**统计
- 各分类**占比饼图**
- 月度消费**趋势折线图**
- 分类**排行榜**

### 5. 数据管理
- 所有数据**本地存储**（不上传到任何服务器，保护隐私）
- 支持**导出数据**（Excel/CSV 格式）
- 支持**数据备份与恢复**

## 🖥️ 技术方案对比

> 以下列出 3 个适合本项目的技术方案。请仔细阅读各方案的说明，选择你最喜欢的一个。

---

### 方案一：Electron + Vue + TypeScript 🟢 推荐

**通俗解释**：就像把网页技术包装成一个桌面软件。用做网站的技术（HTML/CSS/JavaScript）做出一个能在 Windows 和 Mac 上独立运行的软件。

| 项目 | 说明 |
|---|---|
| **优劣势标签** | 🟢 成熟稳定、生态最好 |
| **开发成本** | 中等 |
| **运行性能** | 良好（占用内存约 100-200MB） |
| **安装包大小** | 较大（约 80-150MB） |
| **界面美观度** | ⭐⭐⭐⭐⭐ 非常漂亮，和现代网页一样 |
| **跨平台** | ✅ Windows + Mac 完美支持 |
| **社区资源** | 极其丰富，遇到问题容易找到解决方案 |
| **代表软件** | VS Code、飞书、Discord、Slack |
| **适合场景** | 对界面要求高、需要丰富交互的桌面应用 |

**✅ 优势**：
- 技术非常成熟，社区庞大（VS Code 就是用这个做的）
- 界面可以做得很漂亮，动画流畅
- 有大量现成的 UI 组件库可以直接用，开发快
- 遇到问题时网上能找到很多资料
- 有很多成熟的项目可以参照

**❌ 劣势**：
- 安装包体积较大（但日常使用完全可接受）
- 运行时占用内存稍多（但现代电脑完全够用）

---

### 方案二：Tauri + Vue + TypeScript 🟡 新兴轻量

**通俗解释**：和方案一类似，但更"轻"。它利用操作系统内置的浏览器内核，所以安装包可以做到很小。

| 项目 | 说明 |
|---|---|
| **优劣势标签** | 🟡 轻量快速、新技术 |
| **开发成本** | 中等偏高 |
| **运行性能** | 优秀（占用内存约 50-100MB） |
| **安装包大小** | 很小（约 3-10MB） |
| **界面美观度** | ⭐⭐⭐⭐⭐ 同样可以做得非常漂亮 |
| **跨平台** | ✅ Windows + Mac 完美支持 |
| **社区资源** | 比较新，资源相对少一些 |
| **代表软件** | 一些新型工具类软件 |
| **适合场景** | 追求轻量小巧、对安装包大小敏感 |

**✅ 优势**：
- 安装包非常小（几 MB vs 上百 MB）
- 内存占用更低，运行更轻快
- 技术较新，设计理念现代化
- 安全性好

**❌ 劣势**：
- 技术比较新，社区资源不如 Electron 丰富
- 遇到冷门问题时可能不容易搜到答案
- 部分高级功能可能需要自己多折腾

## 🤔 我的推荐

> 对于你的情况（个人记账、首次做软件、不懂技术），我**推荐方案一（Electron + Vue）**，理由如下：
>
> 1. **最成熟稳定**：这个方案已经被成千上万的应用验证过，出问题的概率最低

## ⚠️ 重要：ELECTRON_RUN_AS_NODE 环境变量问题

> 在开发过程中发现：用户的 Windows 系统级环境变量中设置了 `ELECTRON_RUN_AS_NODE=1`，这导致 Electron 以纯 Node.js 模式运行，而非 Electron 应用模式。
>
> **影响**：
> - `require('electron')` 返回可执行文件路径（字符串）而非 Electron API 对象
> - `process.type` 为 `undefined` 而非 `"browser"`
> - 应用无法正常启动
>
> **解决方案**：项目已在 `scripts/dev.mjs` 和 `scripts/build.mjs` 中自动清除该环境变量。
>
> **永久修复**（可选）：从 Windows 系统环境变量中删除 `ELECTRON_RUN_AS_NODE`，可以彻底解决此问题。
